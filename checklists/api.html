API Pentest Checklist

  • 🔍 Reconnaissance: Identify all accessible endpoints, methods (GET, POST, PUT, DELETE), and documentation (Swagger/OpenAPI).
  • 🔐 Authentication: Test for missing/weak authentication, token leaks, default creds, and improper OAuth/OpenID implementation.
  • 🔑 Authorization: Check for IDOR, privilege escalation, role-based access control bypass, and insecure object references.
  • 📦 Input Validation: Test for injection flaws (SQLi, NoSQLi, command injection), mass assignment, and improper input sanitization.
  • 📤 Rate Limiting: Assess DoS resilience, brute-force protection, and abuse of unauthenticated endpoints.
  • 🧠 Business Logic: Identify pricing manipulation, workflow tampering, and abuse of parameter relationships.
  • 🔎 Error Handling: Ensure consistent and generic error messages, avoid stack traces or internal paths in responses.
  • 🔒 Transport Security: Confirm HTTPS/TLS is enforced, HSTS is used, and certificates are valid and pinned (if applicable).
  • 📄 Data Exposure: Test for PII leakage, sensitive headers, verbose responses, or metadata leaks.
  • 📚 Compliance: Map findings to OWASP API Security Top 10 (2023) and align with industry best practices.
