Android Application Pentest Checklist

  • 📦 APK Analysis: Decompile APK using tools like jadx, MobSF, and analyze AndroidManifest.xml for permissions and exported components.
  • 🔑 Authentication & Authorization: Test for insecure login flows, session hijacking, and broken access controls.
  • 🔐 Data Storage: Review local storage (SharedPreferences, SQLite, files) for sensitive data and enforce proper encryption.
  • 🔍 Static Code Review: Check for hardcoded secrets, insecure APIs, and improper use of WebView or SSL.
  • 🛰 Communication Security: Ensure use of HTTPS/TLS, verify certificate pinning, and test against MITM attacks.
  • 📲 Intent & Component Security: Check for exposed activities, insecure broadcast receivers, and intent injection.
  • 🔧 Reverse Engineering & Debugging: Look for debuggable flags, code obfuscation, tamper detection, and root/jailbreak checks.
  • 🧪 Dynamic Analysis: Use Frida, Xposed, or Burp Suite to intercept and modify runtime behavior and test backend APIs.
  • 🗂 Log & Error Handling: Ensure sensitive data is not logged and exceptions don’t expose internals.
  • 📚 Compliance & Best Practices: Map findings to OWASP MASVS & MSTG standards for mobile application security.
