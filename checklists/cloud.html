Cloud Pentest Checklist

  • ☁️ Cloud Reconnaissance: Identify exposed buckets, services, DNS entries, misconfigured cloud assets, and metadata endpoints.
  • 🔐 Authentication & Identity: Test IAM roles, policies, MFA enforcement, key rotation, and privilege escalation vectors.
  • 🔒 Storage Security: Check S3 buckets, Azure Blobs, or GCP Buckets for public access, misconfigurations, and sensitive files.
  • 📡 Network Security: Assess firewall groups, open ports, insecure endpoints, and internal vs. external exposure.
  • 📦 Service Misconfigurations: Look for excessive permissions, exposed admin consoles, and default credentials.
  • 🔧 Infrastructure as Code (IaC): Review Terraform, CloudFormation, or ARM templates for hardcoded secrets or insecure defaults.
  • 📜 Logging & Monitoring: Ensure centralized logging, CloudTrail/Azure Monitor enabled, and alerts on suspicious activities.
  • 🧠 Business Logic & Abuse: Test for logic flaws in serverless functions, billing abuse, or misused API gateways.
  • 📚 Compliance & Benchmarking: Validate against CIS Benchmarks, AWS Well-Architected Framework, and OWASP Cloud-Native App Security Top 10.
