iOS Application Pentest Checklist

  • 📦 App Package Inspection: Extract IPA, inspect Info.plist, verify permissions, entitlements, and URL schemes.
  • 🔐 Authentication & Session Management: Test login mechanisms, session timeouts, Touch ID/Face ID logic, and token handling.
  • 📲 Local Data Storage: Check for sensitive data in plist, CoreData, NSUserDefaults, or Keychain. Assess encryption and access controls.
  • 🧩 Binary Protections: Check for encryption, PIE, stack canaries, and ensure ASLR, ARC, and code signing are enforced.
  • 🔧 Reverse Engineering: Use tools like Hopper, class-dump, and objection to inspect app internals and runtime behaviors.
  • 🔍 Runtime Manipulation: Leverage Frida or Cycript to test for runtime hooking vulnerabilities and insecure implementation logic.
  • 🛰 Network Communication: Ensure secure transport, validate SSL pinning, and test for MITM resistance and proper certificate handling.
  • 🔄 Inter-Process Communication: Test for insecure custom URL schemes, Universal Links, and unprotected inter-app communication.
  • 📉 Error & Logging: Ensure no sensitive data is leaked in logs, and crash handlers do not expose internal details.
  • 📚 Compliance & MASVS: Map assessment results to OWASP MASVS and Apple security best practices.
