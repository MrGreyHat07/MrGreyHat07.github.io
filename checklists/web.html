Web Application Pentest Checklist

  • 🔍 Information Gathering: Identify server banners, technologies, subdomains, robots.txt, sitemap.xml, and error pages.
  • 🔐 Authentication & Session Management: Test for weak passwords, 2FA bypass, session fixation, and insecure cookie flags.
  • 🔑 Access Control: Verify vertical and horizontal privilege escalation, direct object reference (IDOR), and endpoint access control.
  • 🧪 Input Validation: Test for XSS (Reflected, Stored, DOM), SQLi, SSTI, and command injection.
  • 📤 File Upload: Check for unrestricted file uploads, file type spoofing, and MIME-type validation.
  • 🧠 Business Logic Flaws: Identify flaws in workflows, price tampering, and logic bypasses.
  • 🔌 API Security: Review authentication, rate limiting, sensitive data exposure, and improper object filtering.
  • 📦 Security Headers: Validate implementation of CSP, X-Frame-Options, X-Content-Type-Options, and HSTS.
  • 🔒 SSL/TLS: Ensure valid certificates, HTTP Strict Transport Security, and no weak ciphers or protocols.
  • 📄 Source Code Review: Look for hardcoded secrets, unsafe libraries, and insecure coding practices (optional).
  • 📚 Compliance: Validate against OWASP Top 10 and industry best practices (PCI-DSS, ISO 27001, etc.).
